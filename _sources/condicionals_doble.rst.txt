##################
I què passa sinó…?
##################

"Si tens 18 anys o més llavors ets major d'edat, sinó ets menor"

Podem codificar aquesta nova condició de la següent manera:

.. code-block:: java

    if (edat >= 18) {
        System.out.println("Ets major d'edat"):
    }
    if (! (edat >= 18)) {
        System.out.println("Ets menor d'edat"):
    }

Fixa't que la segona condició és la negació de la primera. Podíem haver-la
expressada simplement com ``edat < 18``.

De manera gràfica, podríem dibuixar-ho com:

.. graphviz::
    :align: center

    digraph {
        inici[shape="box", style=rounded];
        final[shape="box", style=rounded];
        condicio1 [label="edat >= 18", shape="diamond", style=""];
        condicio2 [label="! (edat >= 18)", shape="diamond", style=""];
        instruccio1 [label="escriu «Ets major d'edat»", shape="rectangle", style=""]
        instruccio2 [label="escriu «Ets menor d'edat»", shape="rectangle", style=""]

        inici-> condicio1;
        condicio1 -> instruccio1 [label="cert"];
        condicio1 -> condicio2 [label="fals"];
        instruccio1 -> condicio2;
        condicio2 -> instruccio2 [label="cert"];
        condicio2 -> final[label="fals"];
        instruccio2 -> final;
    }


Si bé això funciona, podem veure-hi dos problemes:

* si ja sabem que no és cert ``edat >= 18``, després
  de la primera pregunta, no caldria tornar-ho a preguntar

* si ja sabem que la primera pregunta resulta certa, i ja hem escrit que és
  major, preguntar si ara és falsa, també és absurd. Ja sabem que no és
  falsa!

El que ens caldria, seria poder expressar el següent diagrama:

.. graphviz::
    :align: center

    digraph {
        inici[shape="box", style=rounded];
        final[shape="box", style=rounded];
        condicio1 [label="edat >= 18", shape="diamond", style=""];
        instruccio1 [label="escriu «Ets major d'edat»", shape="rectangle", style=""];
        instruccio2 [label="escriu «Ets menor d'edat»", shape="rectangle", style=""];

        inici-> condicio1;
        condicio1 -> instruccio1 [label="cert"];
        condicio1 -> instruccio2 [label="fals"];
        instruccio1, instruccio2 -> final;
    }

D'aquesta manera, fixa't que:

* només preguntem un cop si és major d'edat

* el diagrama queda més compacte

Això que hem fet en un diagrama, molts llenguatges de programació,
incloent el Java, ho resolen amb la paraula clau ``else`` que en català es
tradueix com *sinó*.

.. code-block:: java

    if (edat >= 18) {
        System.out.println("Ets major d'edat"):
    } else {
        System.out.println("Ets menor d'edat"):
    }

Amb aquesta nova forma de condicional, podem especificar quines
instruccions a executar quan s'acompleix la condició i quines quan no es
compleix.
